## Introduction

Before a Java program can run, it must be **compiled**.

Compilation is the process of converting human-readable Java source code (`.java`) into machine-independent bytecode (`.class`).

This process is performed by the **Java Compiler (`javac`)**.

---

# Compilation Process

## Command

```
javac Main.java
```

### Example

Source File:

```
public class Main {    
	public static void main(String[] args) {        
		System.out.println("Hello Java");    
	}
}
```

Save as:

```
Main.java
```

Compile:

```
javac Main.java
```

Output:

```
Main.class
```

---

# What Happens During Compilation?

```
Main.java     │     ▼Java Compiler (javac)     │     ▼Main.class
```

The compiler:

- Checks syntax errors
- Checks type errors
- Verifies Java rules
- Converts source code into bytecode

---

# Source Code vs Bytecode

|Source Code|Bytecode|
|---|---|
|Human readable|Machine readable|
|.java file|.class file|
|Written by programmer|Generated by compiler|
|Platform dependent for compilation|Platform independent|

# Why Compilation is Required?

Computers cannot directly understand Java source code.

Java source code:

```
System.out.println("Hello");
```

must first be converted into bytecode that JVM can understand.

---

# Role of Java Compiler

The Java Compiler is part of the JDK.

Responsibilities:

- Syntax checking
- Error detection
- Bytecode generation
- Optimization

# Bytecode in Java

## Definition

**Bytecode** is an intermediate code generated by the Java Compiler after compilation.

It is stored in a `.class` file.

Example:

```
Main.java    ↓javac    ↓Main.class
```

`Main.class` contains bytecode.

---

# What is Bytecode?

Bytecode is neither:

- Source code
- Native machine code

It is an intermediate representation that can be executed by any JVM.

---

# Generation of Bytecode

Source Code:

```
public class Main {    public static void main(String[] args) {        System.out.println("Hello");    }}
```

Compile:

```
javac Main.java
```

Generated:

```
Main.class
```

The `.class` file contains bytecode.

---

# Bytecode Architecture

```
Java Source Code(Main.java)       │       ▼Java Compiler(javac)       │       ▼Bytecode(Main.class)       │       ▼JVM       │       ▼Machine Code       │       ▼Output
```

---

# Benefits of Bytecode

## 1. Platform Independent

The same bytecode can run on:

- Windows
- Linux
- macOS
- Unix

without modification.

Example:

Compile once:

```
javac Main.java
```

Run anywhere with a JVM.

---

## 2. Supports "Write Once, Run Anywhere"

Java's famous principle:

```
Write OnceRun Anywhere (WORA)
```

because of bytecode.

---

## 3. Secure Execution

JVM verifies bytecode before execution.

This helps prevent malicious or invalid code.

---

## 4. Improved Portability

Only JVM needs to be platform-specific.

The bytecode remains the same.

---

# Bytecode vs Machine Code

|Bytecode|Machine Code|
|---|---|
|Generated by javac|Generated by CPU instructions|
|Stored in .class file|Executed directly by processor|
|Platform independent|Platform dependent|
|Executed by JVM|Executed by hardware|
